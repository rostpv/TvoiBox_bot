# Проверка продукта владельцем после запуска

Короткая инструкция для владельца: как убедиться, что новый production работает.

## Перед проверкой

- [ ] Открыт новый production-бот в Telegram.
- [ ] Известен новый mini app domain: `__________`
- [ ] Известен новый API domain: `__________`
- [ ] Тренер находится в Telegram-аккаунте, ID которого указан в `TRAINER_TELEGRAM_ID`.
- [ ] Google Calendar открыт в браузере.

## Быстрая техническая проверка

Открыть в браузере:

- [ ] `https://<api-domain>/health`
- [ ] `https://<app-domain>/`
- [ ] `https://<app-domain>/mini-api/health`

Ожидаемый результат:

- API health отвечает `status: ok` или HTTP 200.
- Mini app открывается.
- `mini-api/health` отвечает HTTP 200.

Результат:

```text
Дата:
API health:
Mini app:
Mini API health:
Комментарии:
```

## Проверка клиента

- [ ] Клиент открывает Telegram-бота.
- [ ] Клиент нажимает `/start`.
- [ ] Появляется кнопка `Открыть mini app`.
- [ ] Mini app открывается внутри Telegram.
- [ ] Клиент видит главное меню.
- [ ] Клиент выбирает доступный слот.
- [ ] Клиент отправляет заявку.
- [ ] Клиент видит, что заявка создана.

Результат:

```text
Клиентский Telegram ID:
Заявка создана:
Что было непонятно:
Ошибки:
```

## Проверка тренера

- [ ] Тренер открывает Telegram-бота.
- [ ] Тренер нажимает `/start`.
- [ ] Тренер открывает mini app или тренерский экран.
- [ ] Тренер видит заявку клиента.
- [ ] Тренер подтверждает заявку.
- [ ] После подтверждения запись появилась в списке тренировок.

Результат:

```text
Тренерский Telegram ID:
Заявка видна:
Заявка подтверждена:
Ошибки:
```

## Проверка уведомлений и календаря

- [ ] Клиент получил Telegram-уведомление после подтверждения.
- [ ] Клиент получил или открыл `.ics` файл.
- [ ] В Google Calendar владельца появилось событие.
- [ ] В событии корректные дата, время, имя клиента и описание.
- [ ] Тренер отменил или перенёс тестовую тренировку.
- [ ] Google Calendar обновился после отмены или переноса.

Результат:

```text
Уведомление клиенту:
.ics:
Google Calendar event:
Отмена/перенос:
Комментарии:
```

## Если что-то не работает

Собрать и передать техническому ответственному:

- [ ] что именно нажимали;
- [ ] время ошибки по Москве;
- [ ] Telegram username/ID пользователя;
- [ ] скриншот ошибки;
- [ ] какой домен открывался;
- [ ] Android/iPhone/desktop;
- [ ] открывалось внутри Telegram или в обычном браузере.

Не пересылать токены и пароли вместе со скриншотами.
